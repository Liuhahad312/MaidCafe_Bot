# 茉晓mox - 咖啡店打工女仆 Bot 说明书

## 0. 茉晓是谁？

茉晓（mox / 茉晚）是一个在咖啡店打工的 17 岁女仆。她活泼可爱，有点冒失，喜欢用颜文字和 emoji，叫她名字就会蹦蹦跳跳地跑过来。

她不是一个冷冰冰的机器人——她是你的打工伙伴、咖啡店同事、占卜小妹。

## 2. 快速开始

### Windows

```
双击 install.bat  →  等待安装完成
编辑 config.yaml  →  填入 DeepSeek 和 Grok 的 API Key
双击 run.bat      →  茉晓启动！
```

### Linux

```bash
chmod +x install.sh run.sh
./install.sh       # 等待安装完成
nano config.yaml   # 填入 API Key
./run.sh           # 茉晓启动！
```

## 3. 配置 NapCatQQ

茉晓使用 OneBot v11 反向 WebSocket 协议与 NapCatQQ 通信。

NapCatQQ 配置中需要添加反向 WS 地址：

```
ws://127.0.0.1:8090/onebot/v11/ws
```

端口号可在 `.env` 文件中修改：

```ini
HOST=127.0.0.1
PORT=8090
```

---

## 4. 配置 config.yaml

```yaml
api:
  deepseek:
    api_key: "sk-你的DeepSeek密钥"
    model: "deepseek-chat"

  grok:
    api_key: "xai-你的Grok密钥"
    model: "grok-2"

limits:
  daily_token_budget_usd: 2.0    # 每日预算上限

misc:
  typo_rate: 0.003               # 正式环境 0.15%，本地测试 0.3
```

## 8. 常见问题

**Q: 启动后没反应？**
A: 检查 NapCatQQ 是否已连接，确认 `.env` 中端口与 NapCatQQ 配置一致。

**Q: API 调用失败？**
A: 检查 `config.yaml` 中 API Key 是否正确填写，确认网络可访问 API 地址。

**Q: 图片修图发不出去？**
A: 确认 NapCatQQ 有发送图片的权限，某些 QQ 协议版本可能不支持本地文件图片。

**Q: 怎么让茉晓更"手癌"？**
A: 修改 `config.yaml` 中 `misc.typo_rate`，0.3=30%概率，0.0015=0.15%概率。
